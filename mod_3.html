3 Sistema de Autenticación

Problema: Implementar un sistema de login que verifique credenciales y controle intentos.

Reglas:

  • Máximo 3 intentos fallidos
  • Usuario y contraseña deben coincidir
  • Bloquear cuenta después del límite
# Sistema de autenticación completo class SistemaLogin: def __init__(self): self.usuarios = {"admin": "123456", "user": "password"} self.intentos = {} self.max_intentos = 3 def login(self, usuario, password): # Inicializar intentos si es necesario if usuario not in self.intentos: self.intentos[usuario] = 0 # Verificar si está bloqueado esta_bloqueado = self.intentos[usuario] >= self.max_intentos if esta_bloqueado: return "Cuenta bloqueada" # Verificar credenciales usuario_existe = usuario in self.usuarios password_correcto = usuario_existe and self.usuarios[usuario] == password if usuario_existe and password_correcto: self.intentos[usuario] = 0 # Resetear intentos return "Login exitoso" else: self.intentos[usuario] += 1 intentos_restantes = self.max_intentos - self.intentos[usuario] return f"Credenciales incorrectas. Resumen de Conceptos Clave

🔢 Operadores Aritméticos

Realizan cálculos matemáticos: +, -, *, /, //, %, **

Uso: Cálculos, contadores, transformaciones de datos

⚖️ Operadores de Comparación

Comparan valores: ==, !=, >, <, >=, <=

Uso: Condiciones, validaciones, decisiones

🧠 Operadores Lógicos

Combinan condiciones: and, or, not

Uso: Lógica compleja, filtros, validaciones

📝 Operadores de Asignación

Asignan valores: =, +=, -=, *=, /=

Uso: Actualizar variables, acumular valores

💡 Tips para el Desarrollo

  • Precedencia: Los operadores tienen un orden de evaluación (**, *, /, +, -, comparación, lógicos)
  • Paréntesis: Usa paréntesis para clarificar expresiones complejas
  • Legibilidad: Divide expresiones largas en múltiples líneas
  • Validación: Siempre valida datos antes de operaciones críticas
  • Debugging: Imprime valores intermedios para depurar lógica compleja
